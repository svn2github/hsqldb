Chapter 1 - Getting Started
Chapter Overview > Client-Server Operation > Accessing A Server From Client Code

A client program can connect to a database instance hosted by a running HSQLDB Server or WebServer using the standard ADO.NET 2.0 mechanism.

First, to obtain and use a connection instance:

    using System.Data.Common;
    ...
    const string ProviderInvariantName = "System.Data.Hsqldb.Client";
    DbFactory factory = DbProviderFactories.GetFactory(ProviderInvariantName);
    using(DbConnection connection = factory.CreateConnection())
    {

Then, to open the connection object:

        /*
        * Alternatively, use "Http" or "Https" to access a WebServer.
        * Use "Hsqls" to specify transport layer security.
        */
        string protocol = "Hsql";
        string host = "localhost";
        int port = 9001;
        string userId = "SA";
        DbConnectionStringBuilder builder = factory.CreateConnectionStringBuilder();
        builder["Protocol"] = protocol;
        builder["Host"] = host;
        builder["Port"] = port;
        builder["User ID"] = userId;
        connection.ConnectionString = builder.ConnectionString;
        connection.Open();
        ...
    }

Note

In the above connection string, there is no mention of the database file; instead, it was specified when running the server.

See Chapter 4 - Configuration And Settings for server configuration details.
