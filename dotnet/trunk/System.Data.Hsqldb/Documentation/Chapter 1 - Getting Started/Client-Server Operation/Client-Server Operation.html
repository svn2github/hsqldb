Chapter 1 - Getting Started
Chapter Overview > Client-Server Operation

One way to expose and access a database instance is through a network interface.

Classically, this mode of operation is referred to as Client-Server.

Client-Server operation provides many benefits, such as maximizing isolation, safety and flexibility.

In Client-Server mode, a database instance runs in a process where a listener daemon waits for network connection requests, typically from other processes, such as from programs running on the same computer or from programs running other computers across the network.

Benefits

  • Database instances are isolated from client machine/process crashes and from external programming errors.
  • An arbitrary number of processes can communicate with a database instance to retrieve or update information

Drawbacks

  • Sigificantly less performant than In-Process operation.
  • May require network configuration / administration.

Note

See Chapter 4 - Configuration And Settings for detailed network server configuration information.
