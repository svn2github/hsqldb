Chapter Overview > In-Process Operation > Mem Protocol

It is possible to run a database instance entirely within random access memory:

    using(DbConnection connection = dbFactory.CreateConnection())
    {
        connection.ConnectionString = "Protocol=Mem;Path=dbname;User ID=SA;";
        connection.Open();
        // ...

Because no information is written to disk, this protocol should typically be reserved for special scenarios.

It is also possible to host a Mem protocol database instance within an HSQLDB Server instance by specifying the protocol on the server command line:

    >System.Data.Hsqldb.Server.exe -database.0 mem:dbname

or in the server.properties file:

    # URI of database
    server.database.0=mem:dbname
    # URL alias of database
    server.dbname.0=scratch

Again, this usage is not common and should be limited to special applications, such as where high-speed transactional processing of transient data is required.

Note

Because information is never written to or read from the file system, the Mem protocol convention is that the first connection to each database instance must be by the "SA" user with no password specified.
