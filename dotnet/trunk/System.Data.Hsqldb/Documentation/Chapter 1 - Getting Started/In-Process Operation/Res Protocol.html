Chapter Overview > In-Process Operation > Res Protocol

Similar to File protocol, except that the database files are resources in an assembly:

    using(DbConnection connection = dbFactory.CreateConnection())
    {
        connection.ConnectionString = "Protocol=Res;Path=pathinsideassembly;User ID=SA";
        connection.Open();
        // ...

Specifically, the Path property of a Res protocol database connection string is the path prefix of the embedded resources that make up the database.

Note

Res protocol is designed specifically to facilitate demonstration operation under Smart ClientTM configurations, where co-locating the database files in the assemblies that make up the application avoids the need for additional security configuration or code signing.
