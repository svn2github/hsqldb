Chapter 1 - Getting Started
Chapter Overview > In-Process Operation

Runs the database engine as part of your application program in the same .NET Virtual Machine.

An In-Process database instance is opened by the first connection (no separate API is required), with the database path specified in the connection string.

Benefits

  • Typically orders of magnitude faster access, as network round-trips are avoided and the data is not converted to and from serial form for transmission over the network.

Drawbacks

  • Ad-hoc out-of-process data access is impossible while the application is running, making development, testing, quality-assurance, etc. less convenient and more expensive.
  • Difficult to control database instance lifecycle:
    • How to avoid or control an open/close cycle per connection?
    • When to close?
    • Who is allowed to close?

To work around the drawbacks, it is possible to run an HSQLDB Server thread in-process with the application and it is possible to control instance lifecycle using the "Shutdown=True" connection string property.

However, it is generally easier and more productive to use an HSQLDB Server instance during elaboration and build cycles, switching to In-Process mode only for formal unit testing, acceptance testing and deployment phases.

There are three main modes of In-Process Operation, each of which is distiguished by a unique connection protocol specifier:

  • File Protocol
  • Res Protocol
  • Mem Protocol
