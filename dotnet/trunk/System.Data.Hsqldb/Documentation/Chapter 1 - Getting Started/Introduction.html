Chapter 1 - Getting Started
Chapter Overview > Introduction

The HSQLDB ADO.NET Data Provider installation contains several components and programs:

  • IKVM Runtime Assembly (IKVM.Runtime.dll)

    An excellent JVM runtime implementation for .NET.

    Among other things, this contains:

    • A Byte Code JIT compiler/verifier:

      Just-in-time compiles Java Byte Code to CIL.

    • An Object model remapping infrastructure:

      Allows Java code to recognize important NET base classes as corresponding Java base classes and visa versa.

    • Managed .NET re-implementations of the native methods in OpenJDK / GNU Classpath.

  • OpenJDK Assembly (IKVM.OpenJDK.ClassLibrary.dll)

    A compiled version of the Java class libraries derived from the OpenJDK class library with some parts filled in with code from GNU Classpath and IcedTea, plus some additional IKVM.NET specific code.

    Note that OpenJDK and GNU Classpath are not part of IKVM.NET, but they are used by IKVM.NET.

  • HSQLDB Embedded RDBMS Engine Assembly (Org.Hsqldb.dll)

    A slightly patched version of the HSQLDB 1.8.0 standard profile jar, recompiled as a managed .NET assembly using the ikvmc.exe static compiler.

  • ADO.NET Data Provider Assembly (System.Data.Hsqldb.Client.dll)

    Contains HSQLDB-specific managed .NET implementations of the abstract base classes found in the System.Data.Common namespace, such as DbConnection, DbCommand, DbDataReader, etc.

  • Common Support Assembly (System.Data.Hsqldb.Common.dll)

    Contains HSQLDB-specific managed .NET classes that provide a bridge between the Java-specific (non-CLSCompliant) Org.Hsqldb.dll assembly and the managed CLR runtime environment.

    In particular, provides the enum types, adaptor, wrapper, and converter classes expected to be either commonly required by or found to be particularly useful in code that needs to interact with the APIs located in the System.Data.Hsqldb.Client, System.Data.Hsqldb.Net or Org.Hsqldb namespaces.

  • Network Server Assembly (System.Data.Hsqldb.Net.dll)

    Contains classes that expose the HSQLDB network database service APIs

  • Server Console Application (System.Data.Hsqldb.Server.exe)

    Allows an HSQLDB Server to be started from the console.

  • Server Service Application (System.Data.Hsqldb.Server.Service.exe)

    Contains a WindowsTM service and service installer for the HSQLDB network database Server

  • WebServer Console Application (System.Data.Hsqldb.WebServer.exe)

    Allows an HSQLDB WebServer to be started from the console.

  • WebServer Service Application (System.Data.Hsqldb.WebServer.Service.exe)

    Contains a WindowsTM service and service installer for the HSQLDB network database WebServer

  • Database Manager WinForms Application (System.Data.Hsqldb.DatabaseManager.exe)

    An interactive SQL GUI.

  • SQL Tool Console Application (System.Data.Hsqldb.SqlTool.exe)

    A batch/interactive SQL console application.

  • Visual Studio 2005 Data Designer Extension Package

    Provides Server Explorer integration with Visual Studio 2005.

  • Visual Studio 2008 Data Designer Extension Package

    Provides Server Explorer integration with Visual Studio 2008.

  • NHibernate Dialect & Driver Assembly

    Allows the data provider to be used with NHibernate for .NET

  • AndroMDA Mapping & Configuration files

    Allows the data provider to be used with AndroMDA and AndroidVS
