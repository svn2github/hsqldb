Chapter 2 - SQL Issues
Chapter Overview > Sequences And Identity > Sequences

An HSQLDB SEQUENCE is an SQL:2003 style external sequence generator created using the syntax:

    CREATE SEQUENCE <sequence-name> <sequence-options>

and its current value is modified using the syntax:

    ALTER SEQUENCE <sequence-name> <sequence-options>

The next value for an external sequence generator is retrieved with the expression:

    NEXT VALUE FOR <sequence-name>

This expression can be used with INSERT, UPDATE and SELECT statements. For example, one can number the rows generated by a SELECT in sequential order using the following syntax:

    SELECT NEXT VALUE FOR mysequence, col1, col2
      FROM mytable
     WHERE ...

Note

The current behavior of HSQLDB external sequence generators is not precisely that defined by in SQL:2003.

If the next value for an external sequence generator is referenced multiple times in a statement, each site receives a new value, whereas SQL:2003 requires each statement-scoped reference to receive the same value.

For now, it is recommended to simply avoid constructs that reference the same sequence generator multiple times within a single statement.  The recommended work-around is to use one or more HSQLDB GLOBAL TEMPORARY tables to hold sequece values and reference these values multiple times within a single statement.

In the future, it is intended to follow precisely the behavior defined in SQL:2003 or later specification.
