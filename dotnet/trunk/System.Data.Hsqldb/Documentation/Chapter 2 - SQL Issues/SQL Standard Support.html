Chapter 2 - SQL Issues
Chaper Overview > SQL Standard Support

HSQLDB supports the dialect of SQL defined by SQL standards 92, 99 and 2003. This means that where a feature of the standard is supported, the HSQLDB syntax is specified by the standards text.

HSQLDB supports many features of SQL92 and 99 up to Advanced Level, and there is support for most of the SQL 2003 Foundation and several of the Optional Features. However, support is by no means complete, so no claims of conformance are made for any level of any standard.

HSQLDB also supports some keywords that are not part of any SQL standard. However, all keywords, including those that are HSQLDB-specific, may be delimited (placed in double quotes) for use as database object identifiers:

    CREATE TABLE mytable(id INTEGER, "POSITION" DECIMAL(10,4))

Similarly, HSQLDB supports some expressions that are not part of any SQL standard, for example:

    SELECT TOP 5 FROM ...
    SELECT LIMIT 0 10 FROM ...
    DROP TABLE mytable IF EXISTS
