Chapter 2 - SQL Issues
Chapter Overview > Types and Arithmetic Operations

Index creation and comparison predicate production is supported for columns of all HSQLDB data types.

However, index creation on LONGVARBINARY, LONGVARCHAR and OTHER columns is deprecated; such indexes will be disallowed in a future version.

All types can be explicitly cast using the CONVERT and CAST SQL functions, but in most cases the built-in implict cast system is sufficient.

Note that in early versions of HSQLDB, there were some artifical limitations on arithmetic operations. For example, it was not possible to insert 10/2.5 into a DOUBLE or DECIMAL column.

All operations are now possible, with the following rules:

  • TINYINT, SMALLINT, INTEGER, BIGINT, NUMERIC and DECIMAL (without a decimal point) are supported integral types and map to sbyte (System.SByte), short (System.Int16), int (System.Int32), long (System.Int64) and (imperfectly) decimal (System.Decimal), respectively.
  • REAL, FLOAT and DOUBLE all map to double (System.Double), although the range of REAL is limited to that of float (System.Single).
  • DECIMAL and NUMERIC both map to decimal (System.Decimal), but are represented internally by java.math.BigDecimal and can have precision and scale far larger than the range accepted by decimal. However, it is possible to submit and retrieve values outside the range of decimal by employing character representation.
