Chapter 4 - Configuration And Settings
Chapter Overview > Connection Settings

The normal method of accessing an HSQLDB database is via the ADO.NET System.Data.IDbConnection interface, using its ConnectionString property to identify the data source.

A full description of each HSQLDB connection string property can be found in the HSQLDB ADO.NET connector API documentation for the System.Data.Hsqldb.Client.HsqlConnectionStringBuilder class.

Basic Connection String Properties

Protocol Host and Port Path
Protocol=Mem; Not Applicable Path=accounts

Lowercase, single-word identifier creates the in-memory database when the first connection is made.

Subsequent use of the same Connection String connects to the existing database instance.
Protocol=File; Not Applicable
  • Path=mydb
  • Path=/opt/db/accounts;
  • Path=C:/data/mydb;

The file path specifies the database file on disk.

The first path example refers to a set of mydb.* files in the working directory of the command used to start the application.

The second and third path examples specify absolute paths on the host machine.
Protocol=Res; Not Applicable Path=resource.path.to.dbname;

Database files are embedded resources in an assembly.
  • Protocol=Hsql;
  • Protocol=Hsqls;
  • Protocol=Http;
  • Protocol=Https;
  • Host=localhost;
  • Host=192.0.0.10;Port=9500;
  • Host=dbserver.somedomain.com
  • Path=/an_alias;
  • Path=/enrollments;
  • Path=/quickdb;

The Host property specifies the IP address or host name of the server and the Port property specifies an optional port number.

The database to which to connect is specified by the Path property and must match a path alias for a database exposed by the server.

A path alias is a lowercase string specified in the server.properties file or on the server command line to refer to an actual database instance hosted by the server.

The following lines in server.properties or webserver.properties specify the database path aliases listed above:

    server.database.0=file:/opt/db/accounts
			server.dbname.0=an_alias

			server.database.1=file:/opt/db/mydb
			server.dbname.1=enrollments

			server.database.2=mem:adatabase
			server.dbname.2=quickdb
