Chapter 4 - Configuration And Settings
Chaper Overview > Configuration Files

HSQLDB relies on a set of configuration files for various settings.

Most of these files are of the java.util.Properties variety, consisting of key=value lines.

In all java.util.Properties variety configuration files, values are treated in a case-sensitive manner.

With the exception of file names and resource URLs, HSQLDB property file values must be specified in lower case.

For example:

server.silent=FALSE

is not recognized, but:

server.silent=false

is recognized.

Core Properties Files

File Name Location Function
server.properties The working directory of the command used to run the Server Settings for an HSQL protocol network database server
webserver.properties The working directory of the command used to run the WebServer Settings for an HTTP protocol network database server
<dbname>.properties The database directory Settings for a specific database instance.

Note

The server and webserver properties files must be created and edited manually.

The database properties file is generated automatically as part of database creation.

Although a database properties file can be edited manually while the database is closed, many database properties can also be altered using SQL commands while the database is open.  When there is a choice, use of SQL is the more portable option.
